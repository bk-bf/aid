Update the documentation file $ARGUMENTS to accurately reflect the current state of the codebase.

Read the file first, then read all config files and source files it references or describes. Identify anything that is outdated, missing, or inaccurate — including stale file paths, line numbers, descriptions of behaviour that has changed, removed features still documented, and new features not yet documented. Then rewrite the affected sections in place. Do not change the document structure or tone unless something is genuinely wrong with it.
